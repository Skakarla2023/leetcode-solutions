# 2540. Minimum Common Value

## Problem Explanation

You are given two sorted integer arrays `nums1` and `nums2`.

Your task is to find the smallest integer that appears in both arrays.

- Return the minimum common value if it exists.
- Otherwise, return `-1`.

### Example

```text
nums1 = [1,2,3]
nums2 = [2,4]

Output: 2
```

Explanation:
- `2` exists in both arrays.
- It is the smallest common value.

---

# Approach

Since both arrays are already sorted, we can solve this efficiently using the **Two Pointer Technique**.

## Steps

1. Initialize two pointers:
   - `i` for `nums1`
   - `j` for `nums2`

2. Compare elements at both pointers:
   - If `nums1[i] == nums2[j]`
     - return the value immediately.
   - If `nums1[i] < nums2[j]`
     - move pointer `i` forward.
   - Otherwise
     - move pointer `j` forward.

3. Continue until one array is completely traversed.

Because the arrays are sorted, the first common element found will automatically be the minimum common value.

---

# Constraints

- `1 <= nums1.length, nums2.length <= 10^5`
- `1 <= nums1[i], nums2[j] <= 10^9`
- Both arrays are sorted in non-decreasing order.

---

# Time Complexity

```text
O(m + n)
```

Where:
- `m = nums1.length`
- `n = nums2.length`

Each pointer moves at most once.

---

# Space Complexity

```text
O(1)
```

No extra space is used.
