A structured DSA practice repository built around **algorithmic pattern recognition** — the most effective way to prepare for technical interviews.

Each solution includes a **brute force approach**, an **optimised solution**, and a **written explanation** of the thought process.

---

## Repository Structure

```
leetcode-solutions/
├── Two_Pointers/
├── Sliding_Window/
├── Binary_Search/
├── Prefix_Sum/
├── Hashing/
├── Stack/
├── Greedy/
├── Backtracking/
├── Dynamic_Programming/
├── Graph/
└── README.md
```

Each folder follows this layout per problem:

```
Sliding_Window/
└── 003-longest-substring-without-repeating/
    ├── Solution.java        ← Optimised solution
    ├── BruteForce.java      ← Brute force solution
    └── NOTES.md             ← Approach, intuition, complexity
```

---

## What's Inside Each Problem

| File | Contents |
|---|---|
| `Solution.java` | Clean, interview-ready optimised code with complexity comments |
| `BruteForce.java` | Initial approach — kept to show the thought progression |
| `NOTES.md` | Problem summary, pattern used, key insight, mistakes made |

---

## Topics Covered

| Topic | Focus Areas |
|---|---|
| **Two Pointers** | Pair problems, palindromes, fast & slow pointer, in-place ops |
| **Sliding Window** | Fixed/variable window, subarrays, substrings |
| **Binary Search** | Rotated arrays, search on answer space, 2D matrix search |
| **Prefix Sum** | Range queries, difference arrays, subarray sums |
| **Hashing** | Frequency maps, anagrams, duplicate detection, O(1) lookups |
| **Stack** | Monotonic stack, next greater element, histogram problems |
| **Greedy** | Interval scheduling, jump problems, local-optimal strategies |
| **Backtracking** | Subsets, permutations, combinations, constraint exploration |
| **Dynamic Programming** | 1D/2D DP, knapsack, LIS, LCS, counting & optimization |
| **Graph** | DFS, BFS, topological sort, Union-Find, shortest path |

---

## Goal

> **Pattern mastery over problem quantity.**

The aim is to build strong enough intuition to recognise the right approach within the first few minutes of seeing a problem — which is exactly what interviews test.
