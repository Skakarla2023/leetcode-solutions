# 876. Middle of the Linked List

## Problem Statement

Given the `head` of a singly linked list, return the middle node of the linked list.

If there are two middle nodes, return the **second middle node**.

### Example 1

Input:
```
1 -> 2 -> 3 -> 4 -> 5
```

Output:
```
3
```

### Example 2

Input:
```
1 -> 2 -> 3 -> 4 -> 5 -> 6
```

Output:
```
4
```

Explanation:

There are two middle nodes (`3` and `4`), so we return the second one (`4`).


## Intuition

Instead of counting all nodes first and then finding the middle, we can use two pointers:

- **Slow Pointer** → moves one step at a time.
- **Fast Pointer** → moves two steps at a time.

Since the fast pointer moves twice as fast as the slow pointer:

- When the fast pointer reaches the end,
- The slow pointer will be at the middle.

This gives us the answer in a single traversal.


## Approach (Two Pointers)

1. Initialize two pointers:
   - `slow = head`
   - `fast = head`

2. Move:
   - `slow` by 1 node.
   - `fast` by 2 nodes.

3. Continue until:
   ```python
   fast == None
   ```
   or
   ```python
   fast.next == None
   ```

4. Return `slow`.




## Code (Java)

```java
class Solution {
    public ListNode middleNode(ListNode head) {
        ListNode slow = head;
        ListNode fast = head;

        while (fast != null && fast.next != null) {
            slow = slow.next;
            fast = fast.next.next;
        }

        return slow;
    }
}
```

  

## Complexity Analysis

### Time Complexity

```
O(n)
```

We traverse the linked list only once.

### Space Complexity

```
O(1)
```

Only two pointers are used.

---

## Key Takeaway

The **Slow and Fast Pointer** technique is a very common linked list pattern.

Whenever you need to:
- Find the middle node
- Detect a cycle
- Find the nth node from the end

Think about using **two pointers moving at different speeds**.
