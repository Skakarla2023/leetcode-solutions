## 141. Linked List Cycle

* Given `head`, the head of a linked list, determine if the linked list has a cycle in it.
* A cycle exists if some node in the list can be reached again by continuously following the `next` pointer.
* Return `true` if there is a cycle, otherwise return `false`.


### Approach

We use **Floyd’s Cycle Detection Algorithm** (Tortoise and Hare).

### Steps

1. Create two pointers:

   * `slow` → moves 1 step
   * `fast` → moves 2 steps
2. Traverse the list.
3. If `slow` and `fast` meet, a cycle exists.
4. If `fast` reaches `null`, no cycle exists.



### Java Code

```java
/**
 * Definition for singly-linked list.
 * class ListNode {
 *     int val;
 *     ListNode next;
 *
 *     ListNode(int x) {
 *         val = x;
 *         next = null;
 *     }
 * }
 */

public class Solution {

    public boolean hasCycle(ListNode head) {

        ListNode slow = head;
        ListNode fast = head;

        // Empty list or single node
        if (head == null || head.next == null) {
            return false;
        }

        // Traverse list
        while (fast != null && fast.next != null) {

            slow = slow.next;
            fast = fast.next.next;

            // Cycle detected
            if (slow == fast) {
                return true;
            }
        }

        // No cycle
        return false;
    }
}
```



### Time and Space Complexity

- **Time Complexity** : O(n)
- **Space Complexity** : O(1)




### Explanation

* The tail connects back to node index `1`.
* So the linked list forms a cycle.



> If two pointers moving at different speeds meet at some point, then a cycle definitely exists.
